<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| resume text → LLM prompt | Текст резюме недоверенный (prompt-injection возможна); подставляется в PROMPT_TEMPLATE. |
| LLM response → ExtractedCandidate | Выход LLM недоверенный; Pydantic-валидация — граница доверия. |
| Extractor → OpenRouter (внешний API) | Сетевой вызов с секретным api_key; внешняя зависимость (timeout/DoS). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | prompt-injection через текст резюме | mitigate | Выход ВСЕГДА проходит `ExtractedCandidate.model_validate` (json_object + строгая схема); инъекция не может выйти за рамки схемы. Провенанс (document_id/model_version) штампуется вызовом, НЕ берётся из ответа LLM — инъекция не подменит провенанс. |
| T-05-05 | Information Disclosure | openrouter_api_key в логах/исключениях | mitigate | Loguru логирует только model/prompt_chars/response_chars — НИКОГДА api_key/prompt/ответ целиком. RuntimeError при отсутствии ключа НЕ печатает значение. OpenAI SDK исключения не содержат ключ в тексте. |
| T-05-06 | Denial of Service | внешний вызов OpenRouter (зависание) | mitigate | `extractor_timeout` (default 60s) передаётся в OpenAI-клиент; вызов оффлоужен в executor — не блокирует event loop. |
| T-05-07 | Repudiation | происхождение извлечённого факта | mitigate | document_id + model_version (фактический id модели, D-03) штампуются на каждом ExtractedCandidate — Фаза 6 переносит на Fact-узлы (критерий #5). |
| T-05-08 | Tampering | повторный provider-side ValidationError | accept | 2-й retry-провал пробрасывает исключение (Фаза 7 пометит документ failed) — не тихий sentinel (D-discretion). |
</threat_model>
