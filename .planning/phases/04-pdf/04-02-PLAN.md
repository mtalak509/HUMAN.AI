| Boundary | Description |
|----------|-------------|
| parser → Neo4j | parser-derived values (document_id, uris) cross into a Cypher write |
| Neo4j availability → parser | external dependency may be down; must degrade gracefully |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering (Cypher injection) | MERGE_DOCUMENT_CYPHER | mitigate | All values passed as bound parameters (`$document_id`, etc.) via session.run — never string-interpolated into Cypher. |
| T-04-06 | Denial of Service | GraphDB.session() when Neo4j down | mitigate | `is_connected` guard before session() (Pitfall 4); outage logs a warning and parse() still returns — no crash, no data loss (files already on disk). |
| T-04-07 | Tampering (duplicate/poisoned nodes) | Document MERGE | mitigate | MERGE on `.id` (SHA-256, document_id_unique constraint) → idempotent; re-parse updates the same node, cannot fork duplicates. |
| T-04-08 | Repudiation | document provenance | accept | ingested_at timestamp set on every MERGE; full provenance (Fact nodes) is Phase 6 scope. |

No high-severity unmitigated threats.
</threat_model>
